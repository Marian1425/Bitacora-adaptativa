Temas vistos en clase.

  • Página web. Es un documento digital accesible en internet que contiene texto, imágenes y otros elementos interactivos, y que se visualiza mediante un navegador.
  • Dominio. “.com” significa “comercial”.
  • Diagrama de flujo.
  • Diagrama de Wireframes.
  • UX. Es como se siente una persona al interactuar con un producto, servicio o sistema.
  • Prototipado. Es como crear un boceto o una versión preliminar de una página web antes de desarrollarla completamente. Baja fidelidad (Wireframes). Alta fidelidad (Wireframes Finos).

HTML (hypertext markup language).

  • Nacimiento. Tim Berners-Lee.
  • Objetivo. Facilitar el intercambio de información mediante la creación de una plataforma basada en hipertexto.
  • Fechas. 1993, 2015.

CSS (Cascading Style Sheets).

  • Nacimiento. Hakon Wium Lie en 1994.
  • Objetivo. Facilitar el diseño visual sin alterar la estructura básica del HTML.

HTML5.

  • Es un lenguaje de marcado utilizado para estructurar el contenido de una página web.

CSS

  • Lenguaje de estilos define como se presenta el lenguaje en la web.

JS (JavaScript)

  • Se utiliza principalmente para crear interactividad o lógica en las páginas web.

Arquitectura de la información.

Se aplica en diversas áreas, como el diseño UX/UI y desarrollo web.

Gráfico de temas previos

DOM(Document Object Model.)

Es una representación en forma de árbol de todo lo que contiene una pagina web:

  • Texto.
  • Etiquetas HTML.
  • Atributos.

DOM Animado.

Modificar las propiedades de los elementos (su posición, tamaño, color, opacidad, etc.) para que cambien con el tiempo y creen la ilusión de movimiento.

DOM de Eventos.

Se encarga de manejar cómo los elementos de una página web reciben e interactúan con eventos como clics, movimientos del mouse, teclas presionadas, etc.

DOM CSS

Parte del DOM que describe y controla los estilos de una página.

¿Para qué sirve "viewport"?

Sirve para indicarle al navegador cómo debe controlar la escala y el diseño de una página web en diferentes dispositivos, especialmente en móviles, para asegurar que el contenido sea visible, legible y se adapte al ancho de cada pantalla.
Es fundamental para el diseño responsivo, ya que le dice al navegador qué ancho debe tener la ventana gráfica del dispositivo y qué escala inicial debe mostrarse el documento, evitando así que el usuario tenga que hacer zoom.
